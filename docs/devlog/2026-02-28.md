## やったこと

### 1. 方向オフセットのバグ修正

**問題**: 外角の球を右方向に打てない (流し打ちが死んでいた)
**原因**: 前回のセッションで `DIRECTION_TIMING_OFFSET_PX = -5` を設定していた。
これは「高速球への反応遅れ補正」のつもりだったが、実際には左方向に常にバイアスがかかっていた。

**修正**: 0 にリバート。シミュレーションで確認:
- 真ん中コース: 方向 -39° 〜 +40° (対称)
- 外角コース: 方向 -25° 〜 +39° (右方向に打てる)

**学び**: 方向制御のような微妙なパラメータは、シミュレーション (simulation.js) で分布を確認してから調整すべき。感覚でいじると簡単に壊れる。

### 3. モバイルタッチ操作

**要件**: スマホでも遊べるようにしたい。UI はシンプルに。

**実装**:
- 画面左 55%: ドラッグでバッター位置調整
- 画面右下 45%: タップでスイング
- READY / GAME_OVER: どこタップでもOK

**座標変換**: CSS でスケールされたキャンバスでもタッチ座標が正しく変換されるように、
`getBoundingClientRect()` で取得した実際のキャンバスサイズで割合計算。

```javascript
_touchToCanvas(touch, canvasRect) {
  return {
    x: (touch.clientX - canvasRect.left) * (CANVAS_WIDTH / canvasRect.width),
    y: (touch.clientY - canvasRect.top) * (CANVAS_HEIGHT / canvasRect.height),
  };
}
```

**タイトル画像の移動**: y=230 → y=80 に上げて、バッターが隠れないようにした。

### 4. フォント改善

**問題**: Bebas Neue の文字間隔が詰まりすぎて読みにくい + 黒い縁取り (strokeText) がダサい

**ユーザーの声**: 「そんな特殊なフォントもこなくていいよ」

**修正**:
- Bebas Neue → `Arial, Helvetica, sans-serif` (システムフォント)
- Google Fonts の `<link>` タグ削除 (外部依存ゼロ)
- 黒縁取り (strokeText 18箇所) → ドロップシャドウ (shadowBlur + shadowColor)
- `ctx.letterSpacing` で大文字テキストの可読性確保

**Before / After**:
```
Before: ctx.strokeStyle = '#000'; ctx.lineWidth = 6; ctx.strokeText(text, x, y);
After:  ctx.shadowColor = 'rgba(0,0,0,0.7)'; ctx.shadowBlur = 8; ctx.fillText(text, x, y);
```

**学び**: Canvas テキスト描画で strokeText による縁取りは太くなりすぎてダサくなりやすい。
shadowBlur の方がモダンで自然な仕上がりになる。ただし shadow はパフォーマンスコストがあるので注意。

### 5. モバイル表示修正 + タッチ重量感

**問題 1**: iPhone で両端が切れる (480px キャンバスが 375px 画面に収まらない)

**修正**: `style.css` に `max-width: 100vw; max-height: 100vh;` 追加。
Canvas は replaced element なので、max-width 制約で高さも自動スケール。
タッチ座標変換は既に `getBoundingClientRect()` ベースなので修正不要。

**問題 2**: バッターのドラッグ操作が軽すぎる (指に直接くっつく)

**修正**: 直接位置セット → ターゲット位置 + 指数スムージング

```javascript
const TOUCH_SMOOTHING = 8;
const t = 1 - Math.exp(-TOUCH_SMOOTHING * dt);
batter.x += (targetX - batter.x) * t;
```

`1 - Math.exp(-k * dt)` はフレームレート非依存の指数スムージング。
k=8 で約 125ms で 63% 到達。60fps でも 30fps でも同じ体感の重さになる。

**学び**: ゲームの「手触り」は lerp/smoothing のパラメータ一つで劇的に変わる。
直接位置セットは「軽い」、指数スムージングは「重い」。
`Math.exp` を使えば dt 依存しない。

### 6. GAME OVER 画面改善

**問題**: タイトルに戻れない (リトライしかできない)

**修正**:
- 「Tap to Retry」テキスト → RETRY / TITLE の2ボタン (角丸矩形)
- タッチ: ボタン座標でヒットテスト
- キーボード: Space = リトライ, Escape = タイトル
- 新メソッド `returnToTitle()`: `resetGame()` + `state = 'READY'`

**ボタン実装**: `ctx.roundRect()` で角丸矩形。半透明 fill + 半透明 stroke で
ゲーム背景が透けて見えるスタイル。RETRY は明るく、TITLE は控えめに。

## 技術メモ

- Canvas の `letterSpacing` プロパティは比較的新しい (Chrome 99+)。古いブラウザでは無視される。
- `ctx.roundRect()` も Chrome 99+ / Safari 15.4+。ポリフィル不要と判断。
- `max-width: 100vw` は Canvas (replaced element) に対して正しく動作する。
  HTML の width/height 属性は内部解像度、CSS は表示サイズを制御。
- Python の `http.server` で no-cache ヘッダを付けた開発サーバーを使用。
  ただしプレビューブラウザ (Playwright) の CSS パーサーが一部プロパティを無視する現象あり
  (実機では問題なし)。
