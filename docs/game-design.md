# HONEY DERBY — ゲームデザインドキュメント

## 概要

「プーさんのホームランダービー」にインスパイアされた、Canvas 2D ベースのバッティングゲーム。
モアイ像が投げるボールを打ち返し、ホームランを量産する。

- キャンバス: 480 x 720px
- フレームワーク: なし (Vanilla JS + Canvas 2D API)
- 入力: キーボード (PC) / タッチ (スマホ)

---

## ゲームフロー

```
READY → COUNTDOWN (3,2,1) → PITCHING → (打撃判定) → RESULT → ...
                                                        ↓
                                              残球 0 → GAME_OVER
                                                        ├─ RETRY → COUNTDOWN
                                                        └─ TITLE → READY
```

- **全 10 球** で 3 本以上 HR を打てば **STAGE CLEAR**
- FOUL は球数にカウントしない (何度でもファウルできる)
- スコア = 全 HR の飛距離合計 (m)

---

## 投球システム

### コース

3 コースからランダム選択。ボールは投手の口 (240, 285) からストライクゾーン中心 (Y=480) へ向かう。

| コース | X座標 | 特徴 |
|--------|-------|------|
| 内角 (inside) | 220 | 引っ張り方向に飛びやすい |
| 真ん中 (middle) | 240 | ニュートラル |
| 外角 (outside) | 260 | 流し打ち方向に飛びやすい |

### ボールの動き

直線的に投手→打者方向へ移動するが、**パースペクティブカーブ** で奥行き感を演出。

```
visualProgress = linearProgress ^ 2.5
```

- 遠い時はゆっくり、近づくと急加速する (3D的遠近感)
- 速度: 350 px/sec
- ボール半径: 4px (遠) → 14px (近) にスケール

### タイミングヒント

ボールがバットの打撃ゾーンに近づくと (60px 手前)、ボールが微かに光る。
打撃タイミングの目安として機能。

---

## バッター操作

### 移動

| 入力 | PC | スマホ |
|------|-----|--------|
| 移動 | 矢印キー | 左半分ドラッグ |
| スイング | スペースキー | 右下タップ |

- 移動速度: 300 px/sec
- 移動範囲: X [110 - 190], Y [520 - 590]
- **タッチ操作は指数スムージング** で重量感を演出

```javascript
// フレームレート非依存の指数スムージング
const TOUCH_SMOOTHING = 8;
const t = 1 - Math.exp(-TOUCH_SMOOTHING * dt);
batter.x += (targetX - batter.x) * t;
```

smoothing = 8 で、約 125ms で残り距離の 63% に到達する「重い」動き。

### スイングアニメーション

11 フレームのスプライトシート。4 つのステートマシンで管理:

```
idle → impact (120ms) → followthrough (100ms) → recovery (280ms) → idle
       バット角: -120° → -360°   → -480°            → -120°
```

- **合計 500ms** の一連のスイング動作
- バット角 -360° (水平) が最適な打撃ポイント
- スイングは一度始まると途中でキャンセル不可

## 打撃判定システム

### 判定フロー

スイングの **impact + followthrough 前半 80ms** の間、毎フレーム判定:

```
1. Y ゾーンチェック (バット接触Y ± 35px 以内?)
2. X ギャップ計算 (バット中心X ± 45px 以内?)
3. スイートスポット距離 → 飛距離算出
4. パワー倍率 (ハートビート) 適用
5. 打球方向計算 (タイミング + コース + バット位置)
6. 判定分類 (HR / HIT / FOUL / STRIKE)
```

### ヒットゾーン

```
        ← 90px (±45px) →
    ┌─────────────────────┐  ↑
    │                     │  70px
    │    ● バット中心      │ (±35px)
    │                     │  ↓
    └─────────────────────┘
```

### スイートスポット

バット先端から 20px 内側にスイートスポットの中心がある。半径 20px。

```
[根本]───────────────[●sweet]─────[先端]
                      ← 20px →
```

- **スイートスポット内**: 飛距離ほぼ最大 (200m ベース)
- **根本側にズレ** (詰まり): ペナルティ大 (0.6 倍)
- **先端側にズレ**: ペナルティ小 (0.2 倍)
- **ゾーン外**: 急速に飛距離減衰

### 飛距離計算

```
baseDistance = 200m × sweetSpotFactor × powerMultiplier
```

- `sweetSpotFactor`: スイートスポットからの距離で 0.0 〜 1.0
- `powerMultiplier`: ハートビートゲージの値 (0.3 〜 1.0)

### 打球方向 (-45° 〜 +45°)

3 つの要素の合算:

| 要素 | 計算 | 影響 |
|------|------|------|
| **タイミング方向** | ボールY - バット接触Y | 早い = 左 (引っ張り), 遅い = 右 (流し) |
| **コースバイアス** | ボールX - ストライクゾーン中心X | 内角 = 左寄り, 外角 = 右寄り |
| **チップバイアス** | バット上の打点位置 | 先端 = 右, 根本 = 左 |

```
directionAngle = timingDirection + courseBias + tipBias
```

- タイミング方向: Y オフセットを 40px で正規化、0.7 乗で増幅
- コースバイアス: 0.5°/px
- チップバイアス: 0.2°/px

### 判定分類

| 判定 | 条件 |
|------|------|
| **FOUL** | 角度 ≥ 40° |
| **HOME_RUN** | 飛距離 ≥ 100m |
| **HIT** (= OUT) | 飛距離 ≥ 30m |
| **STRIKE** | 飛距離 < 30m (当たりが弱い) |

タイミングラベル:
- **Just**: バット接触中心 ± 10px
- **Early**: 中心より上 (ボールが到達前にスイング)
- **Late**: 中心より下 (ボールが通過後にスイング)

## パワーゲージ (ハートビート)

心拍のようにリズミカルに上下するパワーメーター。
ピーク時に打てば最大飛距離、谷間だと 30% まで減衰。

### パラメータ

| 項目 | 値 | 説明 |
|------|-----|------|
| BPM | 72 | 安静時心拍数 |
| Sharpness | 1.0 | ピークの鋭さ (1.0 = 滑らか) |
| Min Power | 0.3 | 谷間での最低倍率 |

### 計算式

```
period = 60000 / BPM (ms)
phase = (elapsed % period) / period     → 0.0 〜 1.0
sine = sin(phase × 2π)                  → -1.0 〜 +1.0
normalized = ((sine + 1) / 2) ^ sharpness → 0.0 〜 1.0
multiplier = 0.3 + normalized × 0.7     → 0.3 〜 1.0
```

### ビジュアル

- 画面左端のイコライザー風バー (幅 12px, 高さ 120px)
- ショッキングピンク (#FF1493) のストライプが充填
- パワー 70% 以上でグロウエフェクト発生

## 打球アニメーション

打撃結果に応じて 3 種類の飛球軌道を描画。

### HOME RUN

```
  ●                        飛距離に応じてアーク高さ変化
 / \                       150px (100m) → 500px (160m+)
/   \
      \  ← ベジェ曲線
       \
        ●  → 遠方へ消える (半径 14→0.5px)
```

- 所要時間: 2500 〜 3500ms (飛距離に比例)
- 方向角度に応じて左右に大きく展開 (±600px)
- ease-out イージング (`progress ^ 0.6`)

### HIT (= OUT)

5 フェーズのバウンド付き外野フライ:

```
Phase 0: ベジェ飛行 → 外野着地 (600ms)
Phase 1: 高バウンド 40px (350ms)
Phase 2: 中バウンド 20px (250ms)
Phase 3: 低バウンド 8px (150ms)
Phase 4: ゴロ転がり (300ms)
合計: 1650ms
```

### FOUL

- 左右画面外へ飛んでいく (800ms)
- 方向に応じて X = -120 or +600 へ
- カーブしながら消える

## モバイル対応

### 表示

- `max-width: 100vw; max-height: 100vh` で画面内にフィット
- Canvas 内部解像度 (480x720) は維持、CSS でスケール
- タッチ座標は `getBoundingClientRect()` で変換

### タッチ操作マップ

```
┌──────────────┬─────────┐
│              │         │
│   ドラッグ    │ (無効)   │  ← 上半分
│  バッター移動  │         │
│   (左55%)    │         │
│              ├─────────┤
│              │ タップ   │  ← 下半分
│              │ スイング  │
│              │ (右45%)  │
└──────────────┴─────────┘
```

### デバイス判定

```javascript
this.isTouchDevice = 'ontouchstart' in window;
```

- PC: 「PRESS SPACE TO START」
- スマホ: 「TAP TO START」

## 開発の履歴とチューニング

### バランス調整の変遷

1. **流し打ち問題**: 外角の球を右方向に打てなかった
   - 原因: `DIRECTION_TIMING_OFFSET_PX` が -5 に設定されていた
   - 修正: 0 に戻して左右対称な方向分布を実現

2. **FOUL ライン拡大**: FOUL 判定角度を調整して打ち分けの幅を確保

3. **TIP バイアス緩和**: 先端打ちの方向偏りが強すぎたので係数を下げた

### UI/UX 改善

1. **フォント変更**: Bebas Neue → Arial (文字間隔が詰まりすぎ + 黒縁取りがダサい)
   - strokeText (黒縁取り) → shadowBlur/shadowColor (ドロップシャドウ)
   - letterSpacing 追加で可読性向上

2. **モバイルタッチ操作**: ドラッグ移動 + タップスイング
   - 直接位置セット → 指数スムージング (重量感)

3. **GAME OVER 画面**: 「Tap to Retry」テキスト → RETRY / TITLE ボタン

4. **タイトル画像**: テキスト「HONEY DERBY」→ ハチミツ木目板の画像に置換
